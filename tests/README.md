Why stdlib only? Two reasons: (a) any future agent on any machine can run the suite without a `pip install` step, and (b) the money paths under test â€” structured data, crawler discovery, conversion attribution â€” should not depend on a fragile test toolchain. Rule of thumb: if a test would require `pip install`, it's out of stdlib scope and belongs in an integration/E2E lane instead.

Each of the nine tests is self-tested against â‰¥6 deliberately-broken inputs (missing fields, drifted origins, orphan mappings, placeholder-image regressions, missing-on-disk assets, orphan `href`s to nonexistent anchors, security-header regressions on nginx locations, Formspree action drift + `required`-strip on the intake form, Google Fonts preload/stylesheet URL drift + comment-embedded `<noscript>` regressions, etc.) â€” never rubber-stamps.
